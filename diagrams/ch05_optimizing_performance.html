Key Concepts: Latency = total cycles for one operation. Issue time = min cycles between starting new operations (pipelining). Capacity = number of functional units of that type. The processor can issue multiple operations per cycle if they use different functional units.

CPE (Cycles Per Element) Improvement

How each optimization reduces the CPE for a combining operation (e.g., summing an array of doubles).

Loop Unrolling

Reduce loop overhead and enable instruction-level parallelism by processing multiple elements per iteration.

Original (1x1)

for (i = 0; i < n; i++) acc = acc OP data[i];

1 element per iteration. Sequential dependency chain on acc.

2x1 Unrolled

for (i = 0; i < n-1; i+=2) acc = (acc OP data[i]) OP data[i+1]; // handle remaining

2 elements per iteration, but still one accumulator. Reduces loop overhead but NOT the critical path.

2x2 Unrolled (2 accumulators)

for (i = 0; i < n-1; i+=2) { acc0 = acc0 OP data[i]; acc1 = acc1 OP data[i+1]; } result = acc0 OP acc1;

Two independent dependency chains! Enables instruction-level parallelism. CPE can be halved.

Throughput Bounds

The CPE is bounded by: max(critical_path_latency / unroll_factor, 1 / throughput). For FP multiply (latency 5, throughput 1/cycle), with 5 accumulators: CPE = max(5/5, 1) = 1.0. You cannot beat the throughput bound.

S = 1 / ((1 - α) + α/k), where α = fraction of time optimized, k = speedup of that part.
