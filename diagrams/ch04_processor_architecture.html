Throughput: ~1 instruction per cycle (ideally). Latency: 5 cycles per instruction. Pipeline registers (F, D, E, M, W) between stages hold intermediate results so each stage can work on a different instruction simultaneously.

Pipeline Hazards

Situations where the next instruction cannot execute in the following clock cycle.

Data Hazard

An instruction depends on the result of a previous instruction still in the pipeline.

addl %eax, %ebx # writes %ebx
subl %ebx, %ecx # reads %ebx - HAZARD!

Solutions: Stalling (insert bubbles) or Forwarding (bypass result from pipeline register)

Control Hazard

A conditional branch is in the pipeline; we don't know which instruction to fetch next.

je target # branch not resolved yet
??? # which instruction?

Solutions: Branch prediction (predict taken/not-taken), squash mispredicted instructions

Load/Use Hazard

Special case: memory load followed immediately by use of loaded value. Cannot forward from Memory stage to Execute.

mrmovl 0(%eax), %ebx # load
addl %ebx, %ecx # use - must stall 1 cycle

Solution: Insert one bubble (stall), then forward from Memory stage

HCL Example

int aluA = [
  icode == IRRMOVL : valA;
  icode in { IOPL, IRMMOVL, IMRMOVL } : valA;
  icode in { IIRMOVL } : valC;
  icode in { ICALL, IPUSHL } : -4;
  icode in { IRET, IPOPL } : 4;
];

HCL uses case expressions (like C switch) to select values based on instruction type. Combinational logic - no state, output depends only on current inputs.
