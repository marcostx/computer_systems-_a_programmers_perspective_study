Every network application is based on the client-server model.

A client is a process that initiates a communication. A server is a long-running process that waits for and responds to requests. A single host can run multiple clients and servers simultaneously. The client and server can be on the same host or different hosts connected by a network.

HTTP Protocol

The Hypertext Transfer Protocol - a simple text-based request/response protocol.

HTTP Request

GET /index.html HTTP/1.1 Host: www.example.com User-Agent: Mozilla/5.0 Accept: text/html Connection: close (blank line = end of headers)

HTTP Response

HTTP/1.1 200 OK Content-Type: text/html Content-Length: 1234 Connection: close <html>...</html>

CGI (Common Gateway Interface)

For dynamic content, the server: (1) fork()s a child process, (2) sets environment variables (QUERY_STRING, etc.), (3) redirects child's stdout to the client socket via dup2(), (4) execve()s the CGI program. The CGI program writes its output to stdout, which goes directly to the client.
