Storage Technology Trends

DRAM density doubles every ~2 years, but access times improve only ~4% per year. CPU clock rates improve ~25% per year. This growing processor-memory gap is the primary motivation for the memory hierarchy with caches.

Locality Patterns

Good locality = good cache performance. Two types of locality drive the memory hierarchy.

Temporal Locality

Recently accessed items are likely to be accessed again soon.

for(i=0; i<n; i++)
  sum += a[i]; // sum accessed n times!

Spatial Locality

Items near recently accessed items are likely to be accessed soon.

for(i=0; i<n; i++)
  sum += a[i]; // stride-1: a[0],a[1],a[2]...

Poor Locality

Large strides or random access patterns thrash the cache.

for(i=0; i<n; i++)
  sum += a[i*stride]; // stride-N: misses!

The Memory Mountain

A visualization of read throughput as a function of spatial locality (stride) and temporal locality (working set size).

Understanding the Mountain

Ridges (temporal locality): When working set fits in L1/L2/L3 cache, throughput is high. Each cache level creates a distinct plateau.

Slopes (spatial locality): Stride-1 access maximizes benefit from cache line prefetch. As stride increases, more cache lines are touched per access, reducing effective throughput.

Valley: Large working set + large stride = worst case. Every access is a main memory access.
