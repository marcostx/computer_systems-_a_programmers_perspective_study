TLB (Translation Lookaside Buffer)

A small, fast cache of recently used page table entries, inside the MMU.

Multi-Level Page Tables

Saves memory by only allocating page table entries for used regions of the address space.

Why Multi-Level?

A single-level page table for 32-bit addresses with 4KB pages needs 220 = 1M entries = 4MB per process. For 64-bit addresses: impossibly large. Multi-level tables only allocate lower-level tables for memory regions actually in use.

Memory Mapping

How the OS maps virtual memory areas to disk objects.

Types of Mapped Objects

TypeDescriptionExamples
Shared ObjectChanges visible to all processes mapping the same objectShared libraries (.so), shared memory
Private Copy-on-WritePrivate copy created only when process writes to pageProgram code/data after fork()
AnonymousNot backed by any file; zero-filled demand pagesHeap, stack, BSS

fork() and execve() with VM

fork(): Creates new page tables for child pointing to same physical pages. Pages marked copy-on-write. Only copied when either process writes.

execve(): Deletes existing VM areas, creates new ones for the new program's .text, .data, .bss, stack. Maps .text and .data from executable file. Does NOT actually load pages until accessed (demand paging).

Implicit Free List Block Format

Header: block size | allocated bit
Payload (if allocated) / Free space
Footer (for coalescing)

The header stores block size (including header+footer) with the low bit indicating allocated (1) or free (0). Coalescing merges adjacent free blocks. Boundary tags (footers) enable coalescing with the previous block in O(1).

Common Memory Bugs in C

Frequent sources of memory-related errors that every C programmer should know.

1. Dereferencing Bad Pointers

scanf("%d", val); // Missing &
scanf("%d", &val); // Correct

2. Reading Uninitialized Memory

int *p = malloc(n*sizeof(int));
// p[i] is NOT zero!
Use calloc() or memset.

3. Stack Buffer Overflow

char buf[64];
gets(buf); // No bounds check!

4. Off-by-One Errors

int a[n];
for(i=0; i<=n; i++) // writes a[n]!

5. Referencing Freed Blocks

free(p);
val = *p; // Dangling pointer!

6. Memory Leaks

p = malloc(size);
p = malloc(size2); // First block leaked!
